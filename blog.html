Question-1: What are the different ways to select an element in the DOM?

getElementById(): Selects a single element by its unique ID.
getElementsByClassName(): Selects all elements with a specific class name.
getElementsByTagName(): Selects all elements with a specific HTML tag name.
querySelector(): Selects the first element that matches a specified CSS selector.
querySelectorAll(): Selects all elements that match a specified CSS selector.

Question-2: What is the difference between innerHTML, innerText, and textContent?

innerHTML:
Gets or sets the HTML markup within an element.
Includes HTML tags, allowing for rich formatting.
Can pose security risks (XSS) if used with unsanitized user input.
It can affect performance, because it can cause the browser to re-render the element.
innerText:
Gets or sets the visible text content of an element.
Considers CSS styling, so hidden elements are ignored.
Tries to represent the text as it would appear rendered in the browser.
It can be slower than textContent, because it requires layout information.
textContent
Gets or sets the text content of an element, including hidden elements.
Ignores CSS styling.
Retrieves all text within an element, regardless of visibility.
It is generally faster then innerText.
In essence:
Use innerHTML when you need to work with HTML markup.
Use innerText when you need to work with the visible text.
Use textContent when you need to work with all text content.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a technique where you attach a single event listener to a parent element, rather than adding individual event listeners to each of its child elements. This approach leverages the concept of event bubbling.
Event Bubbling:
When an event occurs on an element, it "bubbles up" through the DOM tree, triggering event listeners on its parent elements.
How Delegation Works:
Instead of assigning a listener to every child, you assign one listener to the parent. When an event happens on a child, it bubbles up to the parent. The parent's listener then checks the event.target property to determine which child element originated the event.
Benefits:
Performance: Reduces the number of event listeners, improving performance, especially with many child elements.
Dynamic Elements: Handles events on dynamically added child elements without needing to add new listeners.
Simplified Code: Makes event handling code cleaner and more maintainable.

Question-4: What is event bubbling in the DOM?

Event bubbling is a fundamental concept in the Document Object Model (DOM) that describes how events propagate through the hierarchy of elements.
The Process:
When an event occurs on a specific element (the "target" element), that event doesn't necessarily stop there. Instead, it "bubbles up" through the DOM tree, meaning it travels upward to the target element's parent, then to its grandparent, and so on, until it reaches the root of the DOM (typically the window object).
If any of those ancestor elements have event listeners attached to them for that particular event, those listeners will be triggered.
In simpler terms:
Imagine dropping a pebble in a pond. The ripples spread outward. Similarly, an event "ripples" upward through the DOM structure.

Question-5: How do you create, add, and remove elements using JavaScript?

Creating Elements:
document.createElement(tagName): This method creates a new HTML element with the specified tagName (e.g., "div", "p", "span").
Adding Elements:
parentNode.appendChild(childNode): Appends a childNode as the last child of a parentNode. parentNode.insertBefore(newNode, referenceNode): Inserts a newNode before a referenceNode within a parentNode. parentNode.append(childNode1, childNode2, ...): Appends one or more Node or string objects to the last child of the parent. parentNode.prepend(childNode1, childNode2, ...): Prepends one or more Node or string objects to the first child of the parent.
